0.3.0 release train

Comet V3 Milestone Plan

A patch-by-patch path from `0.2.0` to `0.3.0`, focused on cache-aware reads, contract workflows, generated clients, trace propagation, and a clear server-side story.

View Markdown Source
9 patches Usable vertical slices before the final `0.3.0` stabilization cut.
6 workstreams Observability, caching, contracts, codegen, server story, and integration polish.
1 rule Every patch must be release-worthy before the next milestone starts.

North Star

`0.3.0` should make Comet feel like a production networking workflow platform, not only a typed request executor.

  • Cache-aware reads with HTTP revalidation semantics.
  • Deterministic contract testing over the transport and cassette model.
  • A mock-server workflow for demos and tests.
  • Generated request types from a focused OpenAPI subset.
  • Distributed trace propagation across client logs, traces, and servers.
  • A credible server-side transport direction with a documented support boundary.

Patch Train

0.2.1

Release Rails And V3 Planning

Started in `Unreleased` with the milestone plan, roadmap links, changelog coverage, a fresh-client smoke script, and a direct playground test dependency on `HTTPTypes`.

0.2.2

Distributed Trace Propagation

Add `TraceContext`, W3C `traceparent` support, propagation middleware, trace IDs in `RequestTrace`, docs, and playground examples.

0.2.3

Cache Core

Add cache policies, keys, cached response values, cache store protocol, memory store, cache middleware, request options, and trace decisions.

0.2.4

HTTP Revalidation

Parse cache headers, issue conditional requests, merge `304` responses, expose request policies, and trace revalidation decisions.

0.2.5

Persistent Cache And Playground Cache Lab

Add file-backed storage, pruning, stale policies where the model allows, cache demos, cache DocC, and deterministic smoke coverage.

0.2.6

Contract Testing Foundation

Add contract expectations, matching, violation reports, strict transport validation, cassette conversion, and CI-friendly report export.

0.2.7

Mock Server Workflow

Add a `MockServer` facade over contract scenarios, deterministic latency and failure cases, playground demos, and a real-listener decision.

0.2.8

OpenAPI Generator MVP

Add a focused generator or command plugin for JSON OpenAPI documents, generated request types, fixture specs, and compile checks.

0.2.9

Server Direction, Reachability, And TCA Ergonomics

Close the remaining V3 items with a server transport decision, trace compatibility, reachability fit check, and small optional TCA helpers.

0.3.0

Stabilization And Release Cut

Review public API names, update docs, refresh README examples, run full verification, promote to `master`, tag, and publish release notes.
